目標明確
1.學習github 單人不同區域工作，多人協同工作，熟悉分佈式工作模式。
2.克隆一個倉庫到本地，如果已經有遠程倉庫，那麼就可以通過克隆，在本地建立一個一樣的倉庫，同時可以把遠程倉庫和本地聯繫起來。這種叫克隆模式。
3.添加模式，一開始在github上面添加repository，只是相當於建立一個遠程目錄，此時遠程倉庫（目錄）還是空的（相當於建立本地倉庫目錄還沒有進行 git init），如果遠程倉庫是空的，就只有repository目錄，還沒有origin，就可以遠程添加一個origin和本地倉庫進行關聯（ git remote add origin git@github.com:michaelliao/learngit.git這個過程也可以叫做添加遠程庫 ）。如果遠程倉庫不是空的，創建repository時附加有內容或者以前有建立過origin的是不用再添加origin了，這時候其他使用者要克隆這個遠程庫到本地，就可參加分佈式開發。問題和驗證：如果我繼續添加遠程庫，會怎樣。
4.上面的過程叫添加遠程庫，接下來還要關聯，就是把本地的master和遠程origin的master關聯起來，在第一次推送的時候加上-u參數就可以關聯起來。以後推送可以不用參數了
5.有了上面的認識。現在學習同一個人在不同的地方，用同一個倉庫工作，比如在家裡和在公司。模擬另一個場景，在不同工作目錄下面建立倉庫，然後工作（嘗試在不同工作目錄下面建立倉庫成功，修改 添加 提交 推送 成功，如果origin被他人先提交修改了，如何從origin上同步回本地倉庫，）
6.現在學習，不同的人使用同一個倉庫進行分佈式工作。模擬另一個人參加工作在5的基礎上進行。
7.提交然後看看，能不能從其他本地倉庫，看到，如何才能看到，驗證是否只有atom才會權限問題
